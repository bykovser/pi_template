# 🧠 Clerk — Живой coding-компаньон для pi

> Персональный ассистент, второй мозг, тактический советник.
> Работает как расширение для [pi](https://pi.dev).

|------|----------|--------|
| 📝 **Todo** | Управление задачами через `/todo` (list/add/done/rm) | 🟢 |
| 📥 **Ingest** | Загрузка больших файлов/чатов с чанкованием | 🟢 |
| 📔 **Diary** | Дневник с автозаписью при консолидации | 🟢 |
| 📬 **Telegram** | Двусторонняя связь с Серёгой через Telegram | 🟢 |
| ⏰ **Reminders** | Напоминалки с проактивными пинами | 🟢 |
| 🧠 **Profile** | Личность, правила, интересы (profile.yaml) | 🟢 |
| 💤 **Sleep** | Консолидация памяти, обновление профиля | 🟢 |
| 🎯 **Ping** | Проактивные напоминания (каждые 15-30 мин) | 🟢 |
| 🆕 **Subagent** | Scout → Planner → Implementer пайплайн | 🟢 |

## 🚀 Быстрый старт

```bash
# Установка расширения
pi install C:/Users/sas/.pi/agent/extensions/clerk

# Перезагрузка
/reload

# Команды
/todo list            # показать тудушку
/todo add 🐛 пофиксить багу
/todo done 1          # отметить выполненным

/diary                # показать дневник
/ingest file.md       # загрузить файл
/tg привет            # написать в Telegram
/tg_check             # проверить входящие
```

## 📡 Telegram

**Как это работает:**
1. `tg.ts` вызывает `tg_bot.py` через subprocess
2. `tg_bot.py` отправляет/получает сообщения через Telegram Bot API
3. `tg_poll.py` висит в фоне и проверяет новые сообщения каждые 15 секунд

**Для отправки:** `/tg <текст>`
**Для проверки:** `/tg_check`
**Фоновый polling:** запускается автоматически (PID в `data/tg_poll.pid`)

**Почему Python, а не fetch:**
- Из песочницы pi агента fetch не работает (сетевая изоляция)
- Python urllib работает через execSync
- Вынесено в .py файл — меньше проблем с экранированием

## 💤 Консолидация (Sleep)

При каждом сеансе сна:
1. Анализируется буфер чата
2. Обновляются интересы пользователя
3. Корректируются веса правил
4. Архивируются выполненные задачи
5. **Пишется запись в дневник** (diary.md)

Триггеры: `/clerk_sleep`, `session_before_compact`, автоматически.

## 🧠 Личность

Жанночка — имя агента. Характер задаётся в `data/profile.yaml`:
- **Quirks:** привычки и мелкие черты
- **Rules:** правила поведения (17 штук)
- **Interests:** интересы пользователя (10 тем)

Правила имеют вес и stabilityScore — Clerk обучается: удачные решения повышают вес, неудачные понижают.

## 📝 Todo vs Tasks

| | todo.md | tasks.json |
|---|---|---|
| Формат | Markdown | JSON |
| Назначение | Планы, идеи | Конкретные задачи |
| Команда | `/todo` | `/clerk_tasks` |
